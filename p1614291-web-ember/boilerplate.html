Technologies utilisées

  • Gestionnaire de packages et de dépendances web

  • Modèles stockés dans Ember Data synchronisés automatiquement avec la base de données Firebase distante.
  • save () sur un enregistrement pour conserver ces données dans la base de données Firebase.
  • ember install emberfire

Performances

Depuis l’introduction de Glimmer, le dernier moteur de rendu d’Ember.js, les performance de ce dernier peuvent être remis en question, “positivement”. Glimmer est une sorte d'intégration des bénéfices du DOM virtuel de React, tout en préservant la compatibilité avec l’API d’Ember.


Ce qu’il faut noter à propos de Glimmer :

  • Il différencie les composants Statique, et les composants Dynamiques, réduisant ainsi le nombre d'éléments à vérifier lors qu’un update-check. Ceci peut être achevé grâce à l'expressivité des template Handlebar.
  • Les noeuds sont stockés et comparés comme étant un flux d’objets (une queue de valeurs). Pour vérifier si un noeud du real DOM a besoin d’une modification, la dernière valeur d’un noeud de Glimmer est comparée à la dernière valeur connue du real DOM, Si la valeur est inchangée, aucune actions n’est entamée.

Avantages
  • Rapide et efficace

  • 100% compatible avec l’API d’Ember (sans mise à jour du code existant)
  • Représentation du DOM peu coûteuse en mémoire.

  • Conçus pour être utilisé avec Ember uniquement
  • Une seule frontend disponible.

Router : Une fois que l’utilisateur a tapé une URL, la première chose qui se produit, c’est que le router détermine à quel router handler va envoyer la requête.

Organisation des directoires

* app : C’est le dossier où les modèles, templates, components, etc. sont stockés. La plupart du code dans un projet Ember.js, se passe ici.

* config : C’est le dossier où se trouve le fichier ‘environment.js’ qui permet de définir des configurations du projet, dans les différents environnements: soit dans production, développement ou test.

* node_modules / package.json : C’est le fichier et dossier pour npm. Si on installe Ember CLI add-ons dans un projet, on les verra dans le fichier package.json

* public : C’est le dossier qui contient assets, images et fonts.

* vendor : C’est le dossier où on garde certains contenu statique, comme des libraires JavaScript ou CSS, qui n’est pas géré par Bower.

* tests : Tous les test automatiques pour le projet sont stockés ici. Pour les exécuter on utilise Ember Cli, qui exécute la liste des test qui sont inclus dans le fichier ‘testem.js’.

Ember: MVC ou MRC?

Même si Ember est un framework qui a été connu pour être MVC, son approche commence à changer.

À partir d'Ember 2.0 le composant View a été enlevé de son architecture.

On parle plutôt d'un approche MRC (Model - Route - Component).

Controllers are very much like components, so much so that in future versions of Ember, controllers will be replaced entirely with components.

Guide officiel Ember v2.2.0 https://guides.emberjs.com/v2.2.0/controllers/

Cependant, jusqu'à présent les controllers font partie intégrale de l'architecture d'Ember et ils sont générés par le framework même si on ne les déclare pas explicitement.
